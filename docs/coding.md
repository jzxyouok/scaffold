# 团队编码规范

## 简要说明
   项目需遵循惯例优先原则(convention over configuration), 对于很多项目来说，遵从已有的惯例和使用合理的缺省选项大概是最合情合理的做法. 这样要优于屁大点事情都要通过配置文件进行配置.
